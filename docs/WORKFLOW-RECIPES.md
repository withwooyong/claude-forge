# 상황별 워크플로우 레시피

> "이 상황에서 뭘 써야 하지?" - 5가지 상황별 실전 레시피

---

## 레시피 1: 새 기능 만들기

**워크플로우**: `/plan` -> `/tdd` -> `/code-review` -> `/handoff-verify` -> `/commit-push-pr` -> `/sync`

또는 한 번에: `/auto 로그인 기능 만들기`

### 단계별 실제 입력 예시

**1. 계획 세우기**
```
/plan 구글 로그인 기능 만들기
```
AI가 구현 계획을 보여줍니다. "진행해" 라고 답하면 다음 단계로.

**2. 테스트하면서 코딩**
```
/tdd 구글 로그인 기능 구현
```
테스트를 먼저 만들고, 그에 맞는 코드를 작성합니다.

**3. 코드 검사**
```
/code-review
```
보안 문제, 코드 품질을 자동으로 검사합니다.

**4. 최종 검증**
```
/handoff-verify
```
빌드, 테스트, 린트를 한 번에 검증합니다.

**5. 커밋하고 올리기**
```
/commit-push-pr --merge
```
커밋, 푸시, PR 생성, 머지까지 자동으로.

**6. 문서 동기화**
```
/sync
```
프로젝트 문서를 최신 상태로 동기화합니다.

## 레시피 2: 버그 빨리 고치기

**워크플로우**: `/explore` -> `/tdd` -> `/handoff-verify` -> `/quick-commit` -> `/sync`

또는 한 번에: `/auto --mode bugfix 결제 금액이 0원으로 표시되는 버그`

### 단계별 실제 입력 예시

**1. 원인 찾기**
```
/explore payment
```
결제 관련 코드를 탐색하여 구조를 파악합니다.

**2. 테스트로 버그 재현 + 수정**
```
/tdd 결제 금액이 0원으로 표시되는 버그 수정
```
버그를 재현하는 테스트를 먼저 만들고, 수정합니다.

**3. 검증**
```
/handoff-verify --once
```
빠르게 한 번 검증합니다.

**4. 커밋**
```
/quick-commit "fix: 결제 금액 0원 표시 버그 수정"
```
간단한 수정이니 빠르게 커밋합니다.

**5. 문서 동기화**
```
/sync
```
프로젝트 문서를 동기화합니다.

---

## 레시피 3: 코드 정리 (리팩토링)

**워크플로우**: `/refactor-clean` -> `/code-review` -> `/handoff-verify` -> `/commit-push-pr` -> `/sync`

또는 한 번에: `/auto --mode refactor 인증 모듈 정리`

### 단계별 실제 입력 예시

**1. 정리 대상 분석 + 실행**
```
/refactor-clean
```
사용하지 않는 코드, 중복 코드를 찾아서 정리합니다.

**2. 정리 결과 검사**
```
/code-review
```
정리하면서 문제가 생기지 않았는지 검사합니다.

**3. 최종 검증**
```
/handoff-verify
```
빌드, 테스트가 여전히 통과하는지 확인합니다.

**4. 커밋하고 올리기**
```
/commit-push-pr --squash
```
정리 커밋을 하나로 합쳐서 깔끔하게 올립니다.

**5. 문서 동기화**
```
/sync
```
프로젝트 문서를 동기화합니다.

---

## 레시피 4: 보안 점검

**워크플로우**: `/security-review` -> (수정) -> `/handoff-verify --security` -> `/sync`

### 단계별 실제 입력 예시

**1. 보안 스캔**
```
/security-review
```
하드코딩된 비밀번호, SQL 인젝션, XSS 등 보안 취약점을 찾습니다.

**2. 발견된 문제 수정**

보안 리뷰 결과에 따라 수정합니다.

**3. 보안 포함 검증**
```
/handoff-verify --security --effort max
```
최대 깊이로 보안까지 포함하여 검증합니다.

**4. 문서 동기화**
```
/sync
```
프로젝트 문서를 동기화합니다.

## 레시피 5: 사이트 테스트하기

이건 실제 사용자(openclaw_ko)가 발견한 강력한 패턴입니다.

### 실제 입력 예시

**기본 사용**
```
/tdd 이 홈페이지 사용자 입장에서 테스트하고 수정해줘
```

**특정 기능 테스트**
```
/tdd 회원가입 페이지에서 이메일 유효성 검사 테스트하고 수정해줘
```

**모바일 대응 테스트**
```
/tdd 모바일에서 결제 화면이 제대로 보이는지 테스트해줘
```

이 패턴은 TDD 커맨드를 "코딩"이 아닌 "사이트 품질 검사" 용도로 사용하는 것입니다. AI가 사용자 시나리오별 테스트를 만들고, 문제가 있으면 자동으로 수정합니다.

**수정 후 문서 동기화**
```
/sync
```
테스트 및 수정이 완료되면 프로젝트 문서를 동기화합니다.

## 원스톱 자동 모드

위 레시피들이 번거롭다면, `/auto` 커맨드로 전체 파이프라인을 한 번에 실행할 수 있습니다.

```
/auto 로그인 페이지 만들기                              # 새 기능 (기본)
/auto --mode bugfix 결제 금액이 0원으로 표시되는 버그     # 버그 수정
/auto --mode refactor 인증 모듈 정리                    # 코드 정리
```

중간에 멈추지 않고, CRITICAL 보안 이슈에서만 중단합니다.

## 워크플로우 요약 차트

```
새 기능:    /plan -> /tdd -> /code-review -> /handoff-verify -> /commit-push-pr -> /sync
버그 수정:  /explore -> /tdd -> /handoff-verify -> /quick-commit -> /sync
코드 정리:  /refactor-clean -> /code-review -> /handoff-verify -> /commit-push-pr -> /sync
보안 점검:  /security-review -> (수정) -> /handoff-verify --security -> /sync
사이트 QA:  /tdd 이 사이트 사용자 입장에서 테스트하고 수정해줘 -> /sync

원스톱:     /auto [설명]
```
