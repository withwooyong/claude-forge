# 처음 사용자를 위한 가이드

> 설치 완료! 이제 뭘 해야 하지? 이 문서가 답입니다.

---

## 당신의 첫 5분

### 1단계: Claude Code 실행

```bash
cd your-project    # 프로젝트 폴더로 이동
claude             # Claude Code 실행
```

### 2단계: 가이드 시작

```
/guide
```

현재 프로젝트 상태를 자동 진단하고, 다음에 뭘 해야 하는지 안내합니다.

### 3단계: 첫 작업 시작

프로젝트가 준비되었다면, 하고 싶은 작업에 따라:

| 하고 싶은 것 | 입력할 커맨드 |
|-------------|-------------|
| 새 기능 만들기 | `/plan 로그인 기능 만들기` |
| 버그 고치기 | `/tdd 결제 금액이 0원으로 나오는 버그` |
| 코드 정리하기 | `/refactor-clean` |
| 사이트 테스트하기 | `/tdd 이 홈페이지 사용자 입장에서 테스트하고 수정해줘` |

## 자주 쓰는 커맨드 TOP 6

| 커맨드 | 한 줄 설명 | 언제 쓰나 |
|--------|-----------|----------|
| `/plan` | AI가 구현 계획을 세워줍니다. 확인 후 코딩 시작. | 새 기능, 리팩토링 전에 |
| `/tdd` | 테스트 먼저 만들고 코드 작성. 한 단위 작업에 사용. | 기능 구현, 버그 수정 |
| `/code-review` | 방금 작성한 코드를 보안+품질 검사합니다. | 코드 작성 후 |
| `/handoff-verify` | 빌드/테스트/린트를 한 번에 자동 검증합니다. | 작업 마무리 시 |
| `/commit-push-pr` | 검증 완료 후 커밋, 푸시, PR까지 한 번에. | 코드 올릴 때 |
| `/sync` | 프로젝트 문서 동기화. 세션 시작이나 워크플로우 완료 후. | 세션 시작/종료 시 |

## "이건 뭔 말이야?" 용어 사전

개발 용어가 낯설다면, 여기서 찾아보세요.

### TDD (Test-Driven Development)

**비유**: 시험지를 먼저 만들고, 그 다음에 답안을 작성하는 방식입니다.

**기술 설명**: 코드를 쓰기 전에 "이 코드가 이렇게 동작해야 한다"는 테스트를 먼저 작성합니다. 테스트가 통과할 때까지 코드를 구현하고, 통과하면 코드를 정리합니다.

**Claude Forge에서**: `/tdd` 커맨드가 이 전체 과정을 자동으로 진행합니다.

---

### 에이전트 (Agent)

**비유**: 각자 전문 분야가 있는 AI 팀원입니다. 코드 리뷰 전문가, 테스트 전문가, 보안 전문가 등이 있습니다.

**기술 설명**: 특정 역할과 규칙이 설정된 AI 프롬프트입니다. 역할에 맞는 도구와 권한을 가지고 독립적으로 작업합니다.

**Claude Forge에서**: `/plan`, `/tdd`, `/code-review` 등 커맨드를 실행하면 해당 전문 에이전트가 자동으로 호출됩니다.

---

### 핸드오프 (Handoff)

**비유**: "여기까지 했어요" 하고 남기는 인수인계 메모입니다.

**기술 설명**: 현재 작업 상태, 변경 내용, 남은 TODO를 문서로 기록합니다. 다음 세션이나 다른 사람이 이어서 작업할 수 있도록 합니다.

**Claude Forge에서**: `/handoff-verify`가 인수인계 문서 생성 + 자동 검증을 한 번에 처리합니다.

---

### 검증 (Verify)

**비유**: 출고 전 품질 검사입니다. 제품(코드)을 고객(사용자)에게 보내기 전에, 문제가 없는지 전수 검사합니다.

**기술 설명**: 빌드가 되는지, 테스트가 통과하는지, 코드 스타일이 맞는지를 자동으로 확인합니다.

**Claude Forge에서**: `/handoff-verify`가 새로운 AI를 띄워서 편향 없이 처음부터 검증합니다.

---

### 훅 (Hook)

**비유**: 특정 시점에 자동으로 실행되는 안전장치입니다. 현관문을 열면 자동으로 불이 켜지는 것과 같습니다.

**기술 설명**: 파일 저장, 커밋, 세션 시작 등 특정 이벤트가 발생하면 자동으로 실행되는 스크립트입니다.

**Claude Forge에서**: 보안 검사, 코드 품질 체크 등이 훅으로 자동 실행됩니다.

---

### 린트 (Lint)

**비유**: 코드 맞춤법 검사기입니다. 글의 맞춤법을 자동으로 잡아주듯, 코드의 스타일과 잠재적 오류를 잡아줍니다.

**기술 설명**: 코드를 실행하지 않고 텍스트만 분석하여 버그 가능성, 스타일 위반, 보안 문제를 찾아내는 정적 분석 도구입니다.

**Claude Forge에서**: `/handoff-verify`가 빌드, 테스트와 함께 린트도 자동으로 실행합니다.

---

### 커밋 (Commit)

**비유**: 게임의 세이브 포인트입니다. 현재 작업 상태를 저장해두면, 나중에 이 시점으로 돌아올 수 있습니다.

**기술 설명**: Git에서 파일 변경사항을 저장소에 기록하는 행위입니다. 각 커밋에는 "무엇을 왜 바꿨는지" 메시지가 붙습니다.

**Claude Forge에서**: `/commit-push-pr` 또는 `/quick-commit`으로 커밋합니다.

---

### PR (Pull Request)

**비유**: "이거 봐주세요" 하고 올리는 것입니다. 내가 수정한 코드를 팀에 공유하고 검토를 요청합니다.

**기술 설명**: GitHub에서 코드 변경사항을 메인 코드에 합치기 전, 팀원들의 리뷰를 요청하는 기능입니다.

**Claude Forge에서**: `/commit-push-pr`가 커밋, 푸시, PR 생성, 머지까지 한 번에 처리합니다.
